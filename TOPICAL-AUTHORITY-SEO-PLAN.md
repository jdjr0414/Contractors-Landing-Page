## 7. Recommended URL Structure & Internal Linking Pattern

### URL structure

```
/                                    → Homepage
/contractor-[product]                 → Product pillars (working-capital, line-of-credit)
/contractor-[problem]                 → Problem pages (retainage-cash-flow, draw-schedule-cash-flow)
/how-contractors-[action]             → How-to guides (buy-materials-before-getting-paid)
/what-contractors-[situation]        → Situation pages (do-when-invoices-delayed)
/construction-[topic]                 → Construction-specific (equipment-financing, business-loans)
/[product]-vs-[product]               → Comparison pages (equipment-loans-vs-lease)
/contractor-financing/[state]         → State pages (texas, california)
/blog/[slug]                          → Blog articles
/faq                                 → FAQ
/glossary                            → Glossary
/all-funding-options                 → Index
/explore-contractor-funding-options   → CTA hub
```

### Internal linking pattern

1. **Homepage → clusters:** 2–3 links per cluster to pillar pages. Total 12–18 homepage links to key pages.
2. **Pillar → cluster:** Each pillar links to 3–5 cluster pages (problem pages, blog, comparison).
3. **Cluster → cluster:** Problem pages link to 2–3 related problems + 1–2 product pages.
4. **Blog → pillars:** Every blog links to 2–3 relevant pillars + 1 related blog.
5. **Comparison pages:** Link to both products compared; link from both product pillars.
6. **Glossary:** Link from glossary terms to relevant pillars and problem pages.
7. **Breadcrumbs:** Use on all pages for hierarchy and crawlability.

### Link equity flow

```
Homepage (strongest)
  → Pillars (working capital, line of credit, equipment, cash flow problems)
    → Cluster pages (retainage, draws, materials, mobilization, etc.)
      → Blog (supporting long-tail)
```
